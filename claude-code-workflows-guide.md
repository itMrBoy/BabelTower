## Dynamic workflows（动态工作流）

### 功能概述

Dynamic workflows 是 Claude Code 的**多代理任务编排系统**。开启后，Claude 可以将复杂任务分解为多个子任务，分配给不同的子代理（sub-agents）并行或串行执行，最后汇总结果呈现给用户。

### 核心能力

| 能力 | 说明 |
|------|------|
| **并行执行** | 同时启动多个子代理处理独立子任务 |
| **流水线处理** | 前一阶段结果作为下一阶段输入 |
| **动态调度** | 根据中间结果决定后续执行路径 |
| **结果汇总** | 收集各代理结果并整合输出 |

### 使用场景

#### 场景一：大规模代码审查

用户请求审查 PR 的所有改动时，Claude 会启动多个审查代理，分别从不同维度独立审查，最后汇总所有发现的问题。

```
用户：审查这个PR的所有改动
Claude -> 启动多个审查代理（安全性、性能、代码风格）
      -> 各代理独立审查不同维度
      -> 汇总所有发现的问题
```

#### 场景二：多文件重构

需要将项目中的某种模式批量替换时，Claude 会扫描所有文件、分组处理、并行修改、验证结果。

```
用户：将项目中的所有 var 改为 const
Claude -> 扫描所有文件 -> 分组处理 -> 并行修改 -> 验证结果
```

#### 场景三：跨模块调研

分析 Bug 根因时，多个代理分别从不同角度调查，最终综合分析。

```
用户：分析这个 bug 的根因
Claude -> 代理A查日志 -> 代理B查代码 -> 代理C查测试 -> 综合分析
```

### 使用指南

**在对话中触发工作流**

Claude 会自动判断任务复杂度，当任务需要多步骤时会自动启用工作流模式。你也可以明确请求：

```
用工作流方式处理：先搜索所有用到 X 的地方，然后分析影响，最后给出修改方案
```

**观察工作流执行**

开启 Dynamic workflows 后，Claude Code 界面会显示：

- 当前运行的代理列表
- 每个代理的任务状态和进度
- 代理之间的依赖关系图

## Workflow keyword trigger（工作流关键词触发）

### 功能概述

Workflow keyword trigger 是一个快捷命令系统。开启后，你可以在消息开头输入 `/` + 关键词，快速触发预设的特定工作流模式，无需每次都详细描述需求。

### 内置关键词

| 关键词 | 功能 | 使用场景 |
|--------|------|----------|
| `/fix` | 自动诊断并修复代码问题 | 看到报错时直接输入 |
| `/test` | 运行测试并分析失败原因 | 验证代码改动 |
| `/explain` | 详细解释代码逻辑 | 阅读不熟悉的代码 |
| `/commit` | 生成提交信息并执行提交 | 完成改动后 |
| `/review` | 审查代码改动 | PR 审查、自我检查 |
| `/doc` | 生成或更新文档 | 补充代码注释、README |

### 使用场景示例

#### 快速修复

```
/fix 为什么这个测试失败了？
```

Claude 会自动运行测试、定位错误、提出修复方案。

#### 代码解释

```
/explain 这个函数是做什么的？
```

Claude 会详细分析函数逻辑、输入输出、调用关系。

#### 批量审查

```
/review 审查 src/ 目录下所有改动的安全性
```

Claude 会启动安全审查工作流，检查 SQL 注入、XSS 等风险。

### 配置自定义工作流

你可以创建自己的工作流关键词，步骤如下。

#### Step 1：创建工作流定义文件

在项目的 `.claude/workflows/` 目录下创建 YAML 文件：

```yaml
# .claude/workflows/my-audit.yaml
name: security-audit
description: 执行安全审查工作流
steps:
  - name: scan-dependencies
    prompt: "检查 package.json 中的依赖是否有已知漏洞"
  - name: review-inputs
    prompt: "审查所有用户输入处理点，检查注入风险"
  - name: check-auth
    prompt: "验证所有 API 端点是否有适当的身份验证"
```

#### Step 2：配置关键词映射

在 `.claude/settings.json` 中添加映射：

```json
{
  "workflowTriggers": {
    "/audit": "security-audit",
    "/sec": "security-audit"
  }
}
```

配置完成后，输入 `/audit` 或 `/sec` 就会触发安全审查工作流。

### 查看可用工作流

**方式一：输入 `/` 查看列表**

在 Claude Code 中输入 `/` 后按 Tab，会显示所有可用的关键词触发器。

**方式二：查看配置文件**

```bash
# 查看项目级工作流
ls .claude/workflows/

# 查看用户级工作流（全局）
ls ~/.claude/workflows/
```

**方式三：询问 Claude**

```
列出所有可用的工作流关键词
```

## 两个功能的联动关系

```
Dynamic workflows        Workflow keyword trigger
     |                           |
  能力开关                    快捷入口
     |                           |
  决定"能不能"用工作流         决定"怎么快速调用"工作流
     |                           |
     +-----------+---------------+
                 |
           组合使用效果
                 |
  /fix -> 触发"修复工作流" -> 动态调度多个代理
       -> 代理A诊断错误
       -> 代理B搜索相关代码
       -> 代理C生成修复方案
       -> 汇总结果呈现给你
```

- **Dynamic workflows** 是底层能力开关，决定是否启用多代理编排。
- **Workflow keyword trigger** 是上层快捷入口，决定如何快速调用工作流。
- 两者配合使用时，一个关键词即可触发完整的、由多个代理协作的复杂任务。

## 配置建议

在 Claude Code 的 `Config` 页面中：

| 配置项 | 建议值 | 理由 |
|--------|--------|------|
| Dynamic workflows | `true` | 这是 Claude Code 的核心优势功能 |
| Workflow keyword trigger | `true` | 大幅提升常用操作的效率 |

如果你发现 `/` 关键词经常误触发（比如你想输入除法表达式），可以关闭 Workflow keyword trigger，改为用自然语言描述工作流需求。
